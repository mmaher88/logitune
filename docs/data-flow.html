Problem 1: DeviceModel → ProfileEngine (layering violation)

DeviceModel (model layer) directly holds a ProfileEngine* (core layer). Models should not reach into core — AppRoot should mediate.

Problem 2: Duplicate save paths

DeviceModel.setDPI() calls saveProfileToDisk(). AppRoot.saveCurrentProfile() also calls saveProfileToDisk(). Two writers, same file, can clobber.

Problem 3: Getters read hardware, setters write cache

DeviceModel.currentDPI() reads from DeviceManager (hardware). DeviceModel.setDPI() writes to cache. When viewing a non-active profile tab, the UI shows hardware values, not the profile's values.
