# DPI Cycle Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire a first-class `DpiCycle` action end to end: descriptor schema, parser, IDevice getter, pure-function stepping logic, `DeviceSession::cycleDpi()`, `AppController` dispatch, `ActionModel` row, MX Vertical descriptor flip. Every mouse with `adjustableDpi: true` gains a working "DPI cycle" action immediately via a computed fallback ring; MX Vertical ships with a curated factory ring.

**Architecture:** Ring-stepping lives in two pure functions in `src/core/DeviceSession.cpp` (anonymous namespace) so unit tests drive them without any hardware, HID++ transport, or active session. `DeviceSession::cycleDpi()` is a thin wrapper: read `m_currentDPI`, pick next via pure function, call existing `setDPI`. Dispatch follows the same pattern `SmartShiftToggle` uses today: `AppController::onDivertedButtonPressed` (around line 678) adds a `DpiCycle` branch that calls `session->cycleDpi()`.

**Tech Stack:** C++20 / Qt 6 / JSON / GTest. Pure-function extraction keeps test setup minimal.

**Design spec:** `docs/superpowers/specs/2026-04-18-dpi-cycle-design.md`. Read it before Task 1.

```cpp
// ---------------------------------------------------------------------------
// DPI cycle pure helpers
// ---------------------------------------------------------------------------

std::vector<int> DeviceSession::effectiveDpiRing(const std::vector<int> &curated,
                                                 bool adjustableDpi,
                                                 int minDpi, int maxDpi, int step)
{
    if (!curated.empty())
        return curated;
    if (!adjustableDpi)
        return {};
    if (maxDpi <= minDpi || step <= 0)
        return {};
    int mid = (minDpi + maxDpi) / 2;
    // Quantise to step relative to minDpi so the midpoint is a reachable value.
    mid = minDpi + ((mid - minDpi) / step) * step;
    if (mid <= minDpi) mid = minDpi + step;
    if (mid >= maxDpi) mid = maxDpi - step;
    return { minDpi, mid, maxDpi };
}

int DeviceSession::nextDpiInRing(const std::vector<int> &ring, int currentDpi)
{
    if (ring.empty())
        return 0;
    // Find the nearest slot by absolute difference; ties resolve to the
    // lower-indexed slot so a value exactly between two presets still
    // advances predictably.
    std::size_t nearest = 0;
    int bestDiff = std::abs(ring[0] - currentDpi);
    for (std::size_t i = 1; i < ring.size(); ++i) {
        int diff = std::abs(ring[i] - currentDpi);
        if (diff < bestDiff) {
            bestDiff = diff;
            nearest = i;
        }
    }
    return ring[(nearest + 1) % ring.size()];
}
```

If the file does not already `#include <cstdlib>`, add it; `std::abs` on int lives there.

```cpp
void DeviceSession::cycleDpi()
{
    if (!m_connected) {
        qCDebug(lcDevice) << "cycleDpi: not connected, skipping";
        return;
    }
    if (!m_activeDevice) {
        qCDebug(lcDevice) << "cycleDpi: no active device, skipping";
        return;
    }
    auto ring = effectiveDpiRing(m_activeDevice->dpiCycleRing(),
                                 m_activeDevice->features().adjustableDpi,
                                 m_minDpi, m_maxDpi, m_dpiStep);
    if (ring.empty()) {
        qCDebug(lcDevice) << "cycleDpi: no ring available, skipping";
        return;
    }
    const int next = nextDpiInRing(ring, m_currentDPI);
    qCDebug(lcDevice) << "cycleDpi: current" << m_currentDPI << "-> next" << next;
    setDPI(next);
}
```
